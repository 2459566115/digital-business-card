# 作品名称：
——李金燕 - 个人数字名片页

## 作品简介
这是一个基于HTML+CSS开发的现代风格个人数字名片网页，以简历式布局呈现个人基本信息、教育背景、兴趣/技能标签等内容，支持响应式适配（电脑/手机端），视觉风格简洁美观且具备交互微效果，可通过Gitee Pages公开访问。

## 页面主要内容说明
1. 核心信息：姓名（李金燕）、学号（2363170054）、专业（计算机科学与技术）、联系方式（手机号/邮箱）；
2. 个人介绍：超100字的自我介绍，包含学习态度、前端兴趣、生活爱好及未来规划；
3. 特色内容：个性化签名、教育背景（台州学院2023级）、5个兴趣标签、5个技能标签；
4. 多媒体：个人头像（无图片时自动显示渐变字母头像）；
5. 外部链接：指向个人Gitee主页的跳转链接，及个人项目集占位链接。

体现“软件作品”的一点
本作品并非简单的静态文本页面，而是遵循Web开发“结构与样式分离”的工程化思想，具备响应式适配能力（兼容多终端），代码包含交互微效果（标签hover、卡片阴影变化等），文件命名/结构规范、可维护性强，符合软件作品的基本特征（而非仅展示内容的静态页面）。

3. 关键提示词

4. AI主要帮助的部分
快速生成HTML基础结构和CSS样式框架，节省基础代码编写时间；
提供现代UI设计思路（渐变、磨砂玻璃、微交互），提升页面视觉效果；
指导响应式适配的CSS写法（媒体查询）。

5. 对AI结果做的修改
替换AI生成的占位信息为个人真实信息（姓名、学号、专业、获奖情况等）；
调整CSS配色和交互效果（如标签hover颜色、头像尺寸），适配个人审美；
精简AI生成的冗余CSS代码，优化页面加载效率；
补充教育背景模块，完善个人标签（增加追剧、JavaScript等个性化内容）。

6. AI不可靠的一点
AI生成的代码可能存在标签嵌套错误（如多余的</div>），需要手动检查修复；
AI对“Gitee Pages开启步骤”的描述存在版本偏差，需结合实际Gitee界面验证；
AI生成的自我介绍偏模板化，缺乏个人特色，需完全替换为真实表述。
